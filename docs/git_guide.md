# Gitによるバージョン管理ガイド

現在、お使いの環境にはGitがインストールされていません。
以下の手順でインストールと設定を行うことで、ソースコードの履歴管理（バージョン管理）が可能になります。

## 1. Gitのインストール

### 方法A: コマンドでインストール（推奨）
私が提案する以下のコマンドを承認して実行してください。
```powershell
winget install Git.Git
```
※ インストール中に管理者権限の承認ポップアップが表示される場合があります。「はい」を選択してください。
※ インストール後、ターミナルの再起動が必要な場合があります。

### 方法B: 公式サイトからダウンロード
[Git for Windows](https://gitforwindows.org/) にアクセスし、「Download」からインストーラーをダウンロードして実行してください。設定はすべてデフォルトで構いません。

---

## 2. 初期設定（最初の1回だけ）

インストールが完了したら、「誰が変更したか」を記録するためにユーザー名とメールアドレスを設定します。
ターミナルで以下のコマンドを実行します（ご自身のものに書き換えてください）。

```bash
git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"
```

---

## 3. バージョン管理の開始

プロジェクトフォルダで以下のコマンドを実行し、管理を開始します。

1. **初期化**: `.git` フォルダが作成され、管理が始まります。
   ```bash
   git init
   ```

2. **ステージング**: 保存したいファイルを「保存候補」に追加します。
   ```bash
   git add .
   ```
   （`dev` ブランチや `index.html` など、すべての変更を含めます）

3. **コミット**: 変更を確定（保存）します。
   ```bash
   git commit -m "プロジェクトの初期保存"
   ```

---

## 4. 日々の作業サイクル

コードを修正した後は、以下のサイクルを繰り返します。

1. `git add .` （変更をパックする）
2. `git commit -m "XX機能を修正"` （ラベルを付けて保存する）

これで、「いつ」「何を」変更したかの履歴が残り、過去の状態に戻すことも可能になります。
